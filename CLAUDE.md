# Project Overview
This is a NextJS application for AI Frontend Interview Coaching. 

## Key Commands 

- `npm run build` - Build for production
- `npm run typecheck` - Run TypeScript type checking
- `npm test` - Run test suite

## Environment

- Requires a `.env` file at the root with `OPENROUTER_API_KEY="sk-or-..."`.
- `.env.example` is the template. Get a key at <https://openrouter.ai/settings/keys>.
- **Never commit `.env`** — it's gitignored.

## Important Caveats

- **Repo is a fresh scaffold.** This is a stock `create-next-app` output — the c
oaching feature isn't built yet. `app/page.tsx` and `app/layout.tsx` still conta
in template content, and no code reads `OPENROUTER_API_KEY` yet.
- **Scripts vs. reality.** `package.json` currently only defines `dev`, `build`,
 `start`, and `lint`. The `typecheck` and `test` commands above are not wired up
 yet — until they are, type-check with `npx tsc --noEmit`. A `.env.example` temp
late does not exist yet either.
- **Next.js 16 + React 19, App Router.** Per `AGENTS.md`, this Next.js version d
iverges from older releases — read `node_modules/next/dist/docs/` before writing
 framework code.
- **Tailwind CSS v4, CSS-configured.** There is no `tailwind.config.*`. Theme to
kens and `@import "tailwindcss"` live in `app/globals.css` via the `@theme` dire
ctive.
- **Path alias.** `@/*` maps to the repo root (`tsconfig.json`), TypeScript runs
 in `strict` mode.
- **Generated `types/`.** `types/validator.ts`, `types/routes.d.ts`, and `types/
cache-life.d.ts` are auto-generated by Next.js — don't hand-edit them.
